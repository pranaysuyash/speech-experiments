# Prompt Style Guide

**Purpose:** Ensure consistency in prompt creation and usage.

Guidelines:
- Version: Include `**Version:** vX.Y` at the top.
- Purpose: Start with `**Purpose:**` section explaining when to use.
- Structure: Use markdown headings, checklists, and examples.
- Evidence: Require evidence sources and outputs.
- Traceability: Note prompts used in worklog entries and audit artifacts.
- Curation: External prompts must be curated into `prompts/` and added to `prompts/README.md`.

Checklist for new prompts:
- [ ] Version and Purpose sections
- [ ] Clear steps and acceptance criteria
- [ ] Evidence requirements
- [ ] Added to `prompts/README.md`
